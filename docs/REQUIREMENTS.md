# 要件定義書 - Portfolio Website

## 1. プロジェクト概要

### 1.1 プロジェクト名
Kazuya Hibara Portfolio Website

### 1.2 プロジェクトの目的
AIマーケティングエンジニアとしての専門性をアピールし、ソロプレナーや小規模チーム向けのマーケティング自動化サービスへの問い合わせを獲得するポートフォリオサイト。

### 1.3 ターゲットユーザー
| ユーザー層 | 特徴 | ニーズ |
|-----------|------|--------|
| ソロプレナー | 一人でビジネスを運営 | 限られたリソースでマーケティングを自動化したい |
| 小規模チーム | 2-10人程度のチーム | 手作業を減らし、本業に集中したい |
| スタートアップ | 成長フェーズの企業 | スケーラブルなマーケティングシステムが必要 |

### 1.4 プロジェクトスコープ
- ポートフォリオWebサイトの構築
- 多言語対応（日本語・英語）
- 問い合わせフォームの実装
- レスポンシブデザイン

---

## 2. 機能要件

### 2.1 ページ構成

| ページ | パス | 概要 |
|--------|------|------|
| ホーム | `/[locale]` | メインランディングページ、サービス概要、CTA |
| About | `/[locale]/about` | プロフィール、ミッション、経歴 |
| What I Do | `/[locale]/what-i-do` | サービス詳細、ソリューションカテゴリ |
| Cases | `/[locale]/cases` | 事例紹介、ワークフロー例 |
| Process | `/[locale]/process` | 仕事の進め方、4ステッププロセス |
| Contact | `/[locale]/contact` | 問い合わせフォーム |
| Privacy | `/[locale]/privacy` | プライバシーポリシー |

### 2.2 機能一覧

#### FR-001: 多言語対応
- **説明**: 日本語と英語の切り替えが可能
- **優先度**: 高
- **受入条件**:
  - ヘッダーから言語切り替えが可能
  - 現在のページを維持したまま言語を切り替える
  - デフォルト言語は英語

#### FR-002: 問い合わせフォーム
- **説明**: ユーザーが問い合わせを送信できる
- **優先度**: 高
- **受入条件**:
  - 必須項目: 名前、メールアドレス、ビジネス概要
  - 任意項目: 現在の課題、使用ツール、希望連絡方法
  - 送信成功/失敗のフィードバック表示
  - メール通知の送信

#### FR-003: レスポンシブデザイン
- **説明**: モバイル・タブレット・デスクトップに対応
- **優先度**: 高
- **受入条件**:
  - ブレークポイント: sm(640px), md(768px), lg(1024px), xl(1280px)
  - モバイルでハンバーガーメニュー表示
  - タッチデバイスでの操作性確保

#### FR-004: ナビゲーション
- **説明**: サイト内のページ間移動
- **優先度**: 高
- **受入条件**:
  - 固定ヘッダーによるナビゲーション
  - 現在のページをハイライト表示
  - モバイル用ドロワーメニュー

#### FR-005: アニメーション
- **説明**: ユーザー体験向上のためのアニメーション効果
- **優先度**: 中
- **受入条件**:
  - ページロード時のフェードイン
  - ホバー時のインタラクション
  - スムーズなページ遷移

## 3. 非機能要件

### 3.1 パフォーマンス

| 項目 | 要件 | 計測方法 |
|------|------|----------|
| 初期読み込み時間 | 3秒以内 | Lighthouse |
| LCP (Largest Contentful Paint) | 2.5秒以内 | Core Web Vitals |
| FID (First Input Delay) | 100ms以内 | Core Web Vitals |
| CLS (Cumulative Layout Shift) | 0.1以下 | Core Web Vitals |

### 3.2 セキュリティ
- HTTPS通信の強制
- 環境変数による機密情報の管理
- XSS対策（Reactのエスケープ機能活用）
- CSRF対策（フォーム送信時）

### 3.3 可用性
- 99.9%のアップタイム（Vercelホスティング）
- CDNによるグローバル配信

### 3.4 保守性
- TypeScriptによる型安全性
- コンポーネント単位の分離設計
- 国際化ファイルの分離管理

### 3.5 アクセシビリティ
- セマンティックHTML
- 適切なコントラスト比
- キーボードナビゲーション対応
- alt属性の適切な設定

## 4. 外部システム連携

### 4.1 Supabase
- **用途**: バックエンド機能（認証、データベース、ストレージ）
- **現状**: 設定済み（将来の拡張用）

### 4.2 Resend
- **用途**: メール送信API
- **機能**: 問い合わせフォームからの通知メール送信

### 4.3 Vercel
- **用途**: ホスティング、デプロイ
- **機能**: 自動デプロイ、CDN配信、サーバーレス関数

- Edge（最新2バージョン）

### 5.3 デバイスサポート

- デスクトップPC
